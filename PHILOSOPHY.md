# Philosophy

> "The best prompt is the one you don't have to rewrite."

---

## Why This Repo Exists

AI tools are only as good as the context you give them. Without structure, every session starts from zero — re-explaining architecture, naming conventions, and preferences that shouldn't need explaining.

This repo solves that by codifying accumulated knowledge into reusable rules.

---

## Core Principles

### 1. Context Over Commands
Don't tell AI *what* to do on each request. Tell it *who to be* and *what standards to uphold*. A good rules file transforms an AI from a code generator into a project-aware collaborator.

### 2. Rules Are Decisions Made Once
Every rule in this repo represents a decision that was made — often the hard way — in a real project. Write it down once, apply it everywhere.

### 3. Specificity Beats Generality
"Write clean code" is useless. "Use named exports for all components, default export only for page files" is actionable. Every rule should be specific enough that there's no ambiguity in how to apply it.

### 4. Living Documents
Software patterns evolve. A rules file that never gets updated eventually becomes wrong. Every file here has a `Last updated` date. If it hasn't changed in 6 months and you're actively using it, something is wrong.

### 5. Production-Tested Only
No theoretical best practices that sound good but fall apart under real conditions. If a rule is here, it's been used in a real codebase and survived contact with actual requirements.

---

## The Meta-Skill

The real skill isn't writing prompts. It's:

1. **Recognizing patterns** — noticing when you're explaining the same thing to AI for the third time
2. **Abstracting rules** — extracting that explanation into a reusable, generalizable rule
3. **Iterating** — refining rules when they produce wrong outputs or miss edge cases

This is the same skill as good software abstraction, applied to AI collaboration.

---

## What This Is Not

- A prompt collection scraped from the internet
- Theoretical best practices from blog posts
- Rules that only work for toy projects
- A replacement for actually understanding the tools
